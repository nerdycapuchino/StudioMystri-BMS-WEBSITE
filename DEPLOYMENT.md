# Deployment Setup Guide

## One-Time VPS Setup

### 1. SSH into your Hostinger VPS
```bash
ssh root@YOUR_VPS_IP
```

### 2. Run the bootstrap script
```bash
curl -o setup-vps.sh https://raw.githubusercontent.com/YOUR_USER/StudioMystriBMS/main/scripts/setup-vps.sh
bash setup-vps.sh yourdomain.com
```

### 3. Clone the repository onto the VPS
```bash
cd /var/www
git clone https://github.com/YOUR_USER/StudioMystriBMS.git studiomystri
```

### 4. Set up the backend .env
```bash
cp /var/www/studiomystri/backend/.env.production.example /var/www/studiomystri/backend/.env
nano /var/www/studiomystri/backend/.env
# Fill in: DATABASE_URL, JWT secrets, CORS_ORIGIN with your real domain
```

### 5. Run initial DB migration and seed
```bash
cd /var/www/studiomystri/backend
npx prisma migrate deploy
npx prisma db seed
```

### 6. Build both apps for the first time
```bash
cd /var/www/studiomystri/backend && npm ci && npm run build
cd /var/www/studiomystri/frontend && npm ci && npm run build
```

### 7. Configure Nginx
```bash
cp /var/www/studiomystri/nginx/studiomystri.conf /etc/nginx/sites-available/studiomystri
# Edit the file — replace YOUR_DOMAIN with your actual domain
nano /etc/nginx/sites-available/studiomystri
ln -s /etc/nginx/sites-available/studiomystri /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t   # test config — must say "syntax is ok"
systemctl reload nginx
```

### 8. Get SSL certificate
```bash
certbot --nginx -d yourdomain.com -d www.yourdomain.com
# Follow prompts — select option to redirect HTTP to HTTPS
```

### 9. Start the app with PM2
```bash
cd /var/www/studiomystri
pm2 start ecosystem.config.cjs --env production
pm2 save
pm2 list   # verify studiomystri-api shows "online"
```

### 10. Set up daily database backup
```bash
chmod +x /var/www/studiomystri/scripts/backup.sh
# Add to cron: runs every day at 2:00 AM
crontab -e
# Add this line:
0 2 * * * DB_PASSWORD=YOUR_DB_PASS bash /var/www/studiomystri/scripts/backup.sh >> /var/log/studiomystri/backup.log 2>&1
```

## GitHub Actions Setup (do this once)
Add these 3 secrets in GitHub → Repository → Settings → Secrets → Actions:

| Secret Name | Value |
| --- | --- |
| `VPS_HOST` | Your VPS IP address |
| `VPS_USER` | Your VPS SSH username (e.g. root) |
| `VPS_SSH_KEY` | Contents of deploy_key private key |

The private key was generated by `setup-vps.sh` — copy the full output including
`-----BEGIN OPENSSH PRIVATE KEY-----` and `-----END OPENSSH PRIVATE KEY-----`

## After Setup — Every Deploy
Just push to `main` from GitHub Desktop. GitHub Actions will:
1. SSH into VPS
2. `git pull`
3. Build backend + run migrations
4. Build frontend
5. PM2 reload (zero-downtime cluster restart)

Done in ~90 seconds

## Useful Commands
| Task | Command |
| --- | --- |
| View live logs | `make logs` |
| Manual deploy | `make deploy-manual` |
| Check server health | `make status` |
| Restart API | `make restart` |
| Open Prisma Studio | `make studio` (local only) |
| Check Nginx logs | `ssh VPS "tail -f /var/log/nginx/error.log"` |
